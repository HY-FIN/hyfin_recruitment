일단 지금 구현된 admin 탭이 4개가 있는데, 나는 아래와 같이 수정하고 싶어.
1. 대시보드
2. 지원자 관리
3. 면접 관리
4. 메일 발송

대시보드는 아직 내가 아이디어를 더 내야 해서 마지막에 수정하는거로 하고, 지원자관리부터 내가 생각하는 로직을 설명해줄게.

## 지원자 관리 탭
전체 지원자 정보 시트(filttering, sorting 가능)가 오프닝 페이지
-> (평가하러가기 or 지원자 선택) -> 현재 구현된 지원자 관리 탭 등장(지로가기 가능)
지금 구현된 것과 마찬가지로 기본정보/자기소개서/평가로 나뉘고 평가에는 지금 있는 이름 쓰기를 지우고(이미 등록된 id로 admin 페이지에 로그인했기 때문에 불필요), 점수/점수를 준 이유(코멘트)/개인질문을 입력할 수 있는 칸을 만들고, 제출 버튼을 누르면 바로 DB에 저장되도록 구현할거야. 그리고 평가를 완료한 지원자는 전체 지원자 정보 시트에서 바로 확인 가능하도록 구현할거야.

## 면접 관리 탭
면접 구성 캘린더가 오프닝 페이지
-> 캘린더에서 특정 면접 타임(예를들면 면접자 1, 2, 3 26년 8월 20일 19시~19시 30분)을 선택하면 해당 타임의 면접자들의 면접 프로세스(오프닝멘트부터 공통질문, 개인질문, 마무리멘트 순) 등장 여기에 각 지원자들을 평가할 수 있는 칸이 있어서 운영진과 관리자가 평가 후에 저장하면 DB에 적용
-> 관리자가 공통질문 수정을 클릭 하면 모든 면접자들에게 공통적으로 물어볼 공통 질문을 수정할 수 있는 페이지로 넘어감.(3일 면접이라고 가정하고 3세트 준비)
-> 관리자가 면접 캘린더 수정을 하면 면접 일자와 시간 그리고 각 타임별로 어떤 지원자들을 묶을건지 선택할 수 있는 페이지로 넘어감
-> 관리자가 개인질문 수정을 하면 최종적으로 특정 지원자에게 물어볼 개인질문을 수정할 수 있는 페이지로 넘어감

## 메일 발송 탭
서류합불메일, 면접일정메일, 최종합불메일을 발송 할 수 있는 페이지로 구성할거야.
모두 버튼 하나로 일괄전송하게 구현하고 싶은데, 관리자가 선택한 지원자들에게만 발송되도록 구현할거야. (예를들면 서류합불메일은 서류합격자들에게만, 면접일정메일은 면접대상자들에게만, 최종합불메일은 최종합격자들에게만 발송되도록. 그리고 탈락자들에게는 탈락 메일이 발송되도록 구현할거야. 탈락메일은 서류탈락자와 면접탈락자 모두에게 발송되도록 구현할거야. 그리고 메일 발송 후에는 DB에 발송여부가 기록되도록 구현할거야.)
그리고 시간 슬롯을 관리자가 미리 만들어두면 지원자가 거기서 고르는 방식으로 구현할건데, 이 시간 슬롯은 관리자가 만드는게 아니라 내가 미리 만들어둘거야. 아직 정확한 일정은 나오지 않았으니 26년 8월 중순으로 임의의 3일로만들어줘.
